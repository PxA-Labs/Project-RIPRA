# Labels Taxonomy

Issues and PRs are tagged with one **category** label and optionally one or more **cross-cutting** and **tech-specific** labels.

## Category Labels (choose exactly one)

| Label | Description |
|-------|-------------|
| `bug` | Something isn't working |
| `build` | Build system changes (CMake, Docker, scripts) |
| `CI/CD` | CI/CD pipeline changes |
| `documentation` | Improvements or additions to documentation |
| `enhancement` | New feature or request |
| `optimization` | Performance optimization |
| `performance` | Performance-related |
| `refactor` | Code refactoring |
| `repository-hygiene` | Housekeeping |
| `research` | Research investigations |
| `security` | Security-related |
| `testing` | Testing improvements |
| `visualization` | Visualization/dashboard work |

## Cross-Cutting Labels (apply alongside category)

| Label | Description |
|-------|-------------|
| `good first issue` | Good for newcomers |
| `high priority` | High priority |
| `low priority` | Low priority |
| `critical` | Critical priority — must address immediately |

## Tech-Specific Labels (project-specific)

| Label | Description |
|-------|-------------|
| `adaptive-optics` | AO-specific |
| `code-quality` | Code quality |
| `dataset` | Dataset-related |
| `dependencies` | Dependency updates |
| `machine-learning` | ML pipeline |
| `mathematics` | Math/algorithm |
| `robustness` | Robustness |

## Workflow Labels (used for PRs)

| Label | Description |
|-------|-------------|
| `help wanted` | Extra attention needed |
| `invalid` | Not applicable |
| `duplicate` | Already exists |
| `question` | Needs clarification |
| `wontfix` | Will not be worked on |
